ReplicatedRegistry2 Docs

API Reference

Complete API documentation for ReplicatedRegistry.

Types

Table

export type Table = {[any]: any}

A standard Lua table with any keys and values.

FrozenTable

export type FrozenTable = {[never]: any}

A table type indicating read-only access (for type checking).

Filter

export type Filter = (sender: Player?, path: {any}, value: any) -> boolean

A function that validates changes. Returns true to accept, false to reject.

TableChanges

export type TableChanges = {{v: any, p: {any}}}

An array of changes where each change has:

  • v: The new value
  • p: The path to the value (array of keys)

RegistreeInterface

export type RegistreeInterface<T, ReplicateArgs...> = {
    set: (...any) -> (any) -> (),
    get: (...any) -> any,
    incr: (...any) -> (number) -> (),
    replicate: (ReplicateArgs...) -> (),
    full: () -> T & FrozenTable,
}

The proxy interface for convenient table manipulation.

Core Functions

on_recieve

function on_recieve(
    register_key: string,
    fn: (sender: Player?, tbl: Table, changes: TableChanges) -> ()
) -> RBXScriptConnection

Connects a callback to be invoked when changes are received for a registered table.

Parameters:

  • register_key: The unique identifier for the table
  • fn: Callback function receiving:
    • sender: The Player who sent changes (nil if from server)
    • tbl: The updated table
    • changes: Array of applied changes

Returns: RBXScriptConnection that can be disconnected

Example:

local conn = ReplicatedRegistry.on_recieve("player_data", function(sender, tbl, changes)
    print("Received", #changes, "changes")
    for _, change in changes do
        print("Path:", table.concat(change.p, "."), "Value:", change.v)
    end
end)

get_filter

function get_filter(nameToArgs: FilterList) -> Filter

Creates a composite filter from multiple named filters.

Parameters:

  • nameToArgs: Table mapping filter names to their arguments

Returns: A combined filter function

Example:

local filter = ReplicatedRegistry.get_filter({
    player_whitelist = {123456, 789012},
    rate_limit = 10,
    custom = function(sender, path, value)
        return value >= 0
    end
})

set_change_serde

function set_change_serde(
    serializedType: string?,
    ser: ((changes: TableChanges) -> any)?,
    des: ((serialized: any) -> TableChanges)?
)

Sets custom serialization/deserialization for changes sent over network.

Important: Must be called on both client and server with matching functions.

Parameters:

  • serializedType: The type returned by serializer (cannot be "table")
  • ser: Serialization function
  • des: Deserialization function

Example:

-- Using buffer for compact serialization
ReplicatedRegistry.set_change_serde(
    "buffer",
    function(changes)
        return encodeChanges(changes)
    end,
    function(encoded)
        return decodeChanges(encoded)
    end
)

get_changes

function get_changes(registerKey: any) -> TableChanges

Calculates all pending changes since last commit.

Parameters:

  • registerKey: The register key identifier

Returns: Array of changes

commit_changes

function commit_changes(registerKey: any, changes: TableChanges?)

Commits changes to the internal copy, marking them as synchronized.

Parameters:

  • registerKey: The register key identifier
  • changes: Optional pre-calculated changes (if nil, will calculate)

revert_changes

function revert_changes(registerKey: any, changes: TableChanges?)

Reverts the table to its last committed state.
